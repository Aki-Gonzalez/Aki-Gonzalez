SPITE: Bloodloss.
The 6th game project at The Game Assembly. The first with a custom 3D engine: IronWrought.
SPITE is the shared name for this project, all groups use Diablo as a reference game.

About the project


Team: SoftBlob, 13 people (5 programmers, 3 level designers, 4 graphics artists)
Genre: 3D Action RPG
Duration: 14 weeks half time

The first proper 3D-game I’ve been a part of developing.
There is a lot to consider in a 3D-game that I didn’t expect. Surprises that made the group struggle a lot.
The engine, IronWrought was built as the project moved along, in parallel with our 3D programming course.

Main Contributions

click image to view class interface

Debug Tool: LineInstance

Was used to draw colliders and visualize the navigation mesh.
Similar to how we created model. LineInstance holds the data of a Line (geometry data)
and a transform. The line is created in a Factory so that similar Lines are shared.

Modelviewer

Collaboration with Emanuel Niklasson.
Part of the requirements for the project was to create a modelviewer/ showroom.
We didn’t have any GUI-library at our in the engine, instead of adding one we thought it
would be faster to throw something together using the console window for input.
Clunky but got the job done.
We used the std::recursive_directory_iterator to search our project directory for any
fbxes we then filtered the results based on the suffixes the engine used.
It was later also used to preview VFX:es upon request by Nicolas Risberg.

Animations

I took over the work on getting animations up and running.
I wrapped the existing animation controller into a component. Created a pipeline
for it. Spent some time debugging animations made by our graphics artist to figure
out why some animations didn’t work with IronWrought.

Game Object and Component system

GameObject and Component system
Developed with Axel Savage and Alexander Matthäi.
The group decided on creating and using our own component system instead of using
an external solution, like EnTT. Even if ours would be barebones.
We took inspiration from Unity for how we would want our interface to be, which
worked well for what we needed.
