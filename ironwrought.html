IronWrought

IronWrought is the name of the engine that the group, SoftBlob Studios, that I was a part of created durin the second year at The Game Assembly.
It was built as SPITE: Bloodloss was being developed. The engine is written in C++, using DirectX11 for rendering and later on support for PhysX was added. IronWrought is a collaboration between the programmers in the group.

Pipeline and Architecture

Unity was used as a level editor: scenes in Unity were placed into a collection and exported as json and binary files to be read by IronWrought. Prefabs in Unity held graphical assets, scripts and colliders which were translated into GameObjects that IronWrought could use.
IronWrought uses GameObjects and Components to describe entities and objects in a scene. Scenes hold and update these GameObjects as well as a Canvas which holds UI elements such as text and sprites. IronWrought has support for both forward and deferred rendering. A State Stack is used to handle which scene is active.
A Post Master observer pattern is used for communcation betweem objects that do not need to know of each other.
