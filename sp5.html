Doors, Gates and Pressure Plates

The puzzles in Ellah were composed of locks and keys. The 2 locks that we had were doors and gates. Aside from their visual representation being different gates could be locked by scrolls, our story progression items.
For the project the team used Tiled as our level-editor. We used Tiled's templates to bind data to a sprite that we would turn into a GameObject with Components.

Doors, gates and pressure plates were all Components. From Tiled a door or gate were given an UID (unique id) and how many keys it needed to be unlocked. Alexander Matth√§i added the support for gates to check for how many scrolls the player had and how many were needed for it to unlock.
Pressure plates only held which UID they were connected to. On collision with another GameObject, pressure plates checked if the collided object was the player or a movable block then toggling a flag to change its state. To connect the locks and keys I used our PostMaster: doors and gates were Observers that received the state of pressure plates (and other keys) through messages once a state was toggled.

Map Transition

Ellah consisted of a hub world which connected to 4 dungeons, each their own map.
Created in Tiled as a template, the map transition was a Component that held the name of the level it connected to. Because it was odd for the player to enter a dungeon and upon exiting being returned to the beginning of the hub world I added a position to be stored in the component. This position was set by the level designers in Tiled as it gave the safest results.
On collision with the player a message was sent using the PostMaster with a string containing the name for the level to load and the return position was stored in our global state singelton.

Menus - Level Select

All menus were setup in similar fashion with the arrow keys to cycle the options in the menu and return to select an option. Since Ellah was played only using the keyboard this style of navigation felt like a good fit. The menu items consisted of Tga2D CText elements held in an array. The arrow keys incremented and decremented the index that was used for selection. All menus were States that were part of our StateStack that Joakim Gunnari and I setup.

With the level select menu the selected index was used to fetch a string from a separate array which held the name of a level, the string was passed to a LevelLoadState that was pushed to be the active state as the LevelSelectState was popped.
I also created a debug level select in the same menu that used W and S to navigate instead and a std::vector so that it could hold any number of levels.
